1
Transfer files to Sabalcore

In ssClient go to Tools → SFTP File Transfers. Left pane is your PC, right pane is Sabalcore. Navigate to ~/bfr/ on the right. Transfer your .sim file and run.pbs.

2
Submit and monitor
cd ~/bfr
dos2unix run.pbs
qsub run.pbs
watch -n 5 qstat
3
Job states
QQueued — waiting for nodes. No credits charged.
RRunning — credits are being consumed.
CComplete — transfer results via SFTP from ~/bfr/.
4
Live output while running
tail -f ~/.pbs_spool/JOBID.jman.OU

Replace JOBID with your actual job number. Press Ctrl+C to stop.

5
Kill or clear
qdel JOBID.jman
rm -f ~/bfr/*
8
Cluster health checks

Check cluster availability and node load before submitting. Always run upnodes first on production runs.

upnodes
pbsnodes -a | grep -E "loadave|jobs=" | grep -v "^--"
pbsnodes NODE | grep loadave
tracejob JOBID

upnodes shows per-partition availability. pbsnodes shows individual node load. tracejob recovers wall time and resource usage for completed jobs. Replace NODE and JOBID with actual values.

6
Credit & usage

Check your allocation balance and job history.

mybalance
myusage
myusage --summary
myusage --storage

mybalance shows core-hour balance. myusage --summary shows last 12 months. myusage --storage shows disk usage.

7
Inspect a job

Full job details including node allocation, state, and resource usage.

qstat -f JOBID.jman
qstat
ls -la ~/bfr/
cat ~/bfr/BFR_run.eJOBID

The .e file contains stderr — check it first if a job exits unexpectedly. Replace JOBID with your actual job number.
